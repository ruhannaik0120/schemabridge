# E2E PoC Run Artifacts

This folder stores artifacts generated by external E2E QA runs. Each run creates its own folder:

```text
poc_runs/<ticket_id>/
```

Generated ticket run folders are ignored by Git and should not be committed. This `README.md` and `.gitkeep` are kept in Git so the parent folder and its rules remain available.

Never store database credentials, passwords, tokens, connection strings, private keys, or other secrets anywhere in this folder.

An E2E run may contain:

```text
ticket_context.md
qa_plan.md
generated_queries.sql
approval_log.md
run_log.md
execution_result.json
output/report.html
output/report.xlsx
```

The HTML and Excel reports are created only when the user selects those export formats.

## Helper Scripts

From the repository root, initialize a run folder without overwriting existing files:

```powershell
.\.venv\Scripts\python.exe .\scripts\init_poc_run.py ABC-123
```

After `execution_result.json` has been populated by the approved external workflow, export the saved results:

```powershell
.\.venv\Scripts\python.exe .\scripts\export_results.py ABC-123 --format json_only
.\.venv\Scripts\python.exe .\scripts\export_results.py ABC-123 --format html
.\.venv\Scripts\python.exe .\scripts\export_results.py ABC-123 --format excel
.\.venv\Scripts\python.exe .\scripts\export_results.py ABC-123 --format both
```

HTML export uses Python's standard library. Excel export requires the dependency listed in the root `requirements-e2e.txt` file. These helpers only manage local artifacts; they do not call Jira, Atlassian MCP, the database MCP, or any database.
